## 为什么不用 Claude Code 套壳

```diff
- 模型自我认知错乱（"I am Claude" 但你用的是 MiMo）
- 错误信息全是 Anthropic 术语
- 配置目录冲突（一会儿 Claude 一会儿 MiMo，开俩工具就乱）
- 看不到 MiMo 的 thinking 字段
- 看不到 prompt 缓存命中数据
- 违反 Claude Code 的 ToS（用 Anthropic 工具调非 Claude 模型）
+ 原生 MiMo 体验 · 模型自我认知正确
+ 所有 UI / 命令 / 日志 MiMo 化
+ 独立配置 `~/.mimo/`，跟 Claude Code 并存互不污染
+ 实时渲染 thinking + 缓存命中
+ 完全合规：原创 + 用 MiMo 官方公开协议
```

## 永远免费 · 没有 Pro / 企业版

mimo-tui 是 MIT 开源项目。**不收费、不卖数据、不投广告**。

为什么免费？
- 工具本身是给 MiMo Token Plan 用户做的，已经付过钱给小米了
- 我们希望 MiMo 生态有一个像 Claude Code 一样能用的官方级体验
- 卖给 MiMo 用户既不合理也不必要

每个用户用自己的 MiMo Token Plan API key（跟 Claude Code 用各自 `ANTHROPIC_API_KEY` 一样）。

> "MiMo" 是小米注册商标 · mimo-tui 是社区开源项目 · 与小米无官方关联
