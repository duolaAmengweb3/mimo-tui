## 核心能力，不靠“能接 API”充门面

<CardGrid>
  <Card title="完整 agent loop" icon="rocket">
    Anthropic Messages API 风格循环、streaming SSE、最多 32 步工具迭代，够支撑真正的复杂任务。
  </Card>
  <Card title="8 个内置工具" icon="seti:powershell">
    read / write / edit / shell / glob / grep / web_fetch / todo，基础生产力开箱就齐。
  </Card>
  <Card title="子 agent 并行" icon="random">
    `task` 工具最多并行 10 个子 agent，把长任务拆成可管理的执行单元。
  </Card>
  <Card title="MCP 双协议" icon="puzzle">
    同时支持 stdio 和 HTTP，现成 MCP server 装上就能用，不用二次适配。
  </Card>
  <Card title="Skills 兼容" icon="open-book">
    兼容 Anthropic Skills 格式，已有 `.claude/skills/` 可以直接迁入。
  </Card>
  <Card title="三平台沙箱" icon="setting">
    macOS Seatbelt、Linux Landlock、Windows Job Objects 默认启用，安全边界不是事后补丁。
  </Card>
  <Card title="三模式审批" icon="bars">
    Plan、Agent、Auto 三种模式对应不同风险偏好，终端里就能切换。
  </Card>
  <Card title="智能区域路由" icon="document">
    CN / SGP / AMS 自动测延迟，区域 key 不可用时自动 fallback。
  </Card>
</CardGrid>

## 跟同类最大的差别，不是 UI，是产品判断

| | 套壳 Claude Code | XIAOMI-MiMo-code | **mimo-tui** |
|---|---|---|---|
| 架构来源 | Anthropic 工具壳 | 逆向方案 | **从零实现** |
| 模型身份 | “I’m Claude” | prompt 修补 | **原生 MiMo 身份** |
| 配置目录 | 易冲突 | 独立 | **`~/.mimo/` 完全独立** |
| thinking / cache | 部分不可见 | 不完整 | **完整暴露** |
| MCP 能力 | 有 | 部分 | **stdio + HTTP** |
| 合规性 | 有 ToS 风险 | 高风险 | **原创 + MIT** |
| 可维护性 | 受上游约束 | 脆弱 | **自主演进** |

## 免费，而且会长期免费

mimo-tui 是 MIT 开源项目。没有 Pro、没有企业版墙、没有广告，也不绑定托管服务。

- 用户自己持有 MiMo Token Plan API key，付费关系直接在用户和模型服务之间。
- 工具层保持原创和轻量，目标是把 MiMo 生态补齐到“真正可日用”的开发体验。
- 如果以后继续升级，重点也应该放在稳定性、协议兼容和工作流设计，而不是做花哨收费层。

> "MiMo" 是小米注册商标。mimo-tui 是社区开源项目，与小米无官方关联。
