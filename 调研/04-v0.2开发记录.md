# mimo-tui v0.2 开发记录

> 时间：2026-05-19 到 2026-05-20
> v0.1 → v0.2 单次对话内完成

## 这次新增的能力

### 1. MCP 客户端（约 500 行 Rust）

完整 stdio transport：

- `crates/mcp/jsonrpc.rs` — JSON-RPC 2.0 wire types
- `crates/mcp/protocol.rs` — MCP 协议类型（initialize / tools/list / tools/call）
- `crates/mcp/stdio.rs` — 子进程 transport（含 reader task + oneshot 回复通道 + kill_on_drop）

集成进 agent：

- `core::McpHub::init()` 启动时 spawn 所有 `~/.mimo/mcp.json` 配置的 server
- 每个 MCP tool 以 `mcp__<server>__<tool>` 命名空间注册到 `ToolRegistry`
- 模型自动获得 MCP 工具，无需修改 agent loop

**真实验证**：用 `@modelcontextprotocol/server-filesystem` 跑通，agent 自主选择 `mcp__fs__read_text_file` 读文件。

### 2. Skills 系统（约 250 行 Rust + 4 个测试）

完全兼容 Anthropic Skills 格式：

- YAML frontmatter + markdown body 解析
- `triggers` 数组显式匹配 / `description` 关键字 fallback
- 加载 `~/.mimo/skills/`、`./.claude/skills/`、`./.mimo/skills/` 三处
- 首轮匹配的 skill 内容自动注入 system prompt

**真实验证**：一个写 "always add type hints + docstring" 的 skill 让 mimo 第一次写 Python 就符合规范。

### 3. CI / 多平台编译

3 个 GitHub Actions workflow：

- `test.yml` — 三平台 cargo test + clippy + fmt
- `release.yml` — tag 触发，6 个 target 自动 cross-compile（mac arm/x64 + linux x64/arm64/musl + windows x64），自动 release + sha256
- `website.yml` — website/ 变更自动 Cloudflare Pages 部署

### 4. npm 跨平台 postinstall

- `install.js` 检测平台/架构 → 从 GitHub Releases 下载对应二进制 → sha256 验证 → 解压
- `bin/mimo.js` 透传 stdin/stdout 给原生二进制
- 支持 5 个目标平台

### 5. 一键 install.sh

- 部署在 `mimo-tui.pages.dev/install.sh`
- 自动检测 darwin / linux gnu / linux musl / windows
- 装到 `/usr/local/bin` 或 fallback `~/.local/bin`

## 下一步（v0.3 / v1.0 路线）

按优先级：

1. **流式渲染**：当前 agent 用非流式 `messages()` API；改用 `messages_stream_raw` + 逐 token 渲染，UX 大幅提升
2. **完整 ratatui TUI**：rustyline REPL → 全屏分窗口（输入 / 输出 / 状态栏 / 用量看板）
3. **子 agent 编排**：基于现有 Tool 抽象，加 `task` 工具调度并行子任务
4. **三平台沙箱**：macOS Seatbelt / Linux Landlock / Windows Job Objects
5. **MCP HTTP / SSE transport**：补全 v0.2 只有 stdio 的限制
6. **多模型 fallback**：Token Plan 耗尽时切到备用 endpoint（DeepSeek / Qwen 等 OpenAI 兼容）

## 经验总结

1. **Skills 系统价值远超想象**：50 行 Rust + frontmatter 解析 = 让用户能完全控制 agent 行为。这是 Claude Code 火爆的核心机制之一。
2. **MCP 集成关键是 wire format 细节**：camelCase 是个小坑但耽误半天。建议未来加 mock MCP server 单元测试。
3. **跨平台二进制不是难题**：GitHub Actions + `cross` crate，配好 workflow 就跑。runner 等队列要 5-10 分钟。
4. **dogfood 是最好的 demo**：让 mimo 自己写自己的公告稿，比任何人工写的更可信。
