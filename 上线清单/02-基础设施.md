# 02 · 基础设施

> 目标：所有账号 / 仓库 / 权限就位
> 估时：0.5 天
> 依赖：无

## 2.1 GitHub

- [ ] 创建 GitHub 组织（推荐 `mimo-tui` 或 `mimotui`）
- [ ] 创建主仓库 `mimo-tui/mimo-tui`（公开、MIT）
- [ ] 创建 Homebrew tap 仓库 `mimo-tui/homebrew-mimo-tui`
- [ ] 创建官网仓库 `mimo-tui/website`（或合在主仓库的 `website/` 目录）
- [ ] 启用 GitHub Container Registry（ghcr.io）
- [ ] 配置 GitHub Actions secrets：
  - [ ] `NPM_TOKEN`（npm 发布用）
  - [ ] `CARGO_REGISTRY_TOKEN`（crates.io 发布用）
  - [ ] `CLOUDFLARE_API_TOKEN`（Pages 部署用）
  - [ ] `CLOUDFLARE_ACCOUNT_ID`

## 2.2 npm

- [ ] 注册 npm 账号（如还没有）
- [ ] 申请 `mimo-tui` 包名（先 publish 一个空版本占位）
- [ ] 启用 2FA
- [ ] 生成 automation token，存到 GitHub secrets

## 2.3 crates.io

- [ ] 用 GitHub 登录 crates.io
- [ ] 占位发布 `mimo-tui` crate（v0.0.0）
- [ ] 生成 API token

## 2.4 Cloudflare

- [ ] 创建 Cloudflare 账号（如还没有）
- [ ] `wrangler login`（已完成 ✅）
- [ ] 创建 Cloudflare Pages 项目 `mimo-tui`
- [ ] 拿到 `mimo-tui.pages.dev` 子域名
- [ ] 生成 API token（具备 Pages:Edit 权限），存到 GitHub secrets

## 2.5 Docker Hub / GHCR

- [ ] GHCR 用 GitHub 自带，无需额外注册
- [ ] 主仓库 Settings → Actions → Workflow permissions → Read and write
- [ ] 确认 `packages:write` 权限给到 workflow

## 2.6 域名（可选 - 完全 ¥0 不做）

- [ ] **跳过**：直接用 `mimo-tui.pages.dev`
- [ ] 未来如果要：在 Cloudflare Registrar 买 `mimo-tui.dev`

## 2.7 第三方账号

- [ ] Discord 服务器创建（社群用）
- [ ] X / 推特账号 `@mimotui`（推广用，可选）
- [ ] Reddit 账号（投 r/programming / r/rust）

## 验收

- [ ] 所有 secrets 都在 GitHub 仓库 Settings → Secrets 里
- [ ] `wrangler whoami` 显示已登录
- [ ] `npm whoami` 显示已登录
- [ ] `cargo login` 已配置
- [ ] 主仓库可以 push 代码（哪怕只是 README）
