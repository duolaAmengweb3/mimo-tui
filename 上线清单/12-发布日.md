## 12.1 发布顺序（按时间）

### T-1 day（前夜）
- [ ] 最终自检（11 重跑一遍）

- [ ] 社交稿件全部 ready，链接占位

### T 0:00（发布日开始）

- [ ] CI 自动跑 release workflow
- [ ] 等 release workflow 跑完（30–60 min）

### T +30min（自动化部分完成后）

## 12.2 必备发布物料

- [ ] GitHub Release notes（含视频链接）
- [ ] 知乎长文（2000+ 字，含截图）
- [ ] V2EX 帖子（500+ 字，简练）
- [ ] X thread（5–10 条推文）
- [ ] HackerNews 标题 + URL 测试
- [ ] Reddit 标题（每个 sub 量身定制）
- [ ] 微信公众号（含动图 / 视频）
- [ ] 邮件模板（给小米官方 + 给社区 KOL）

## 12.3 危机预案

- [ ] **CI 挂了** → fallback：手动构建上传
- [ ] **npm 名字被抢** → fallback：用 `@<org>/mimo-tui`
- [ ] **首日 P0 bug** → 立即 hotfix v1.0.1
- [ ] **小米发声明** → 提前准备好回应模板（强调原创 / MIT / 不收费）
- [ ] **被某博主黑** → 不撕逼，发文澄清事实

## 12.4 发布日成功指标（24h 内）

- ⭐ 至少 100 stars
- 📥 至少 200 npm 下载
- 💬 至少 1 篇外部博客 / 帖子讨论
- 🐛 P0 bug = 0
- 🔥 至少 1 个 trending 出现（GitHub trending Rust / HackerNews 首页等）
